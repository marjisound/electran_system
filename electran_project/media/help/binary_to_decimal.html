Hexadecimal-binary conversion

Conversion between numbers in different bases

The methods for converting a number between bases, e.g. turning 010110012 to 8910 can be derived from the power series representation of a number.

The power series representation for 2 different bases, b and k, is:-

I = s0b0 + s1b1 + s2b2 + s3b3 + .... + smbm = t0k0 + t1k1 + t2k2 + t3k3 + ..... + tnkn

The development of the conversion algorithms to convert I from base b to base k is best seen by re-writing the power series representations:-

I = t0 + k(t1 + k(t2 + .. + k(tn-1 + ktn))) = s0 + b(s1 + b(s2 + .. + b(sm-1 + bsm)))


Method 1:

This converts a positive integer, I, from its representation in base k to its representation in base b, when addition and multiplication are available in base b.

Basically this requires the calculation of the symbols smsm-1......s2s1s0 in base b from the (known) symbols of I in base k, i.e. tntn-1...t4t3t2t1t0.

The method starts by taking k and the symbols t0, t1, t2,.... tm, converting each into its representation in base b and inserting them in the power series formulation I in terms of k

I = t0 + k(t1 + k(t2 + .... + k(tn-1 + ktn))),

valid independent of the base.

This yields, after multiplication and addition in base b, the representation of I in base b.

This operation can be done sequentially one symbol at a time: set sum to 0; add tn to sum, multiply sum by k, add tn-1 to sum, multiply sum by k, etc. The algorithm does not need to know how many significant symbols there are in either the base b or base k representation.

Example: Convert 1101112 to base 10

k = 2, b = 10, kb = 210
Stage 0: sum10 = 0
Stage 1: t5 = 12 = 110 : sum10 = sum * 2 + t5 = 0 * 2 + 1 = 1
Stage 2: t4 = 12 = 110 : sum10 = sum * 2 + t4 = 1 * 2 + 1 = 3
Stage 3: t3 = 02 = 010 : sum10 = sum * 2 + t3 = 3 * 2 + 0 = 6
Stage 4: t2 = 12 = 110 : sum10 = sum * 2 + t2 = 6 * 2 + 1 = 13
Stage 5: t1 = 12 = 110 : sum10 = sum * 2 + t1 = 13 * 2 + 1 = 27
Stage 6: t0 = 12 = 110 : sum10 = sum * 2 + t0 = 27 * 2 + 1 = 55
Done: 1101112 = 5510

Example: Convert BCDE16 to base 10

k = 16, b = 10, kb = 1610
Stage 0: sum10 = 0
Stage 1: t3 = B16 = 1110 : sum10 = sum * 16 + t3 = 0 * 16 + 11 = 11
Stage 2: t2 = C16 = 1210 : sum10 = sum * 16 + t2 = 11 * 16 + 12 = 188
Stage 3: t1 = D16 = 1310 : sum10 = sum * 16 + t1 = 188 * 16 + 13 = 3021
Stage 4: t0 = E16 = 1410 : sum10 = sum * 16 + t0 = 3021 * 16 + 14 = 48350
Done: BCDE162 = 4835010

Method 2:

This converts a positive integer, I, from base b to base k, where division and remainder are available in base b.

Basically this requires the calculation of the symbols tntn-1........t4t3t2t1t0 in base k from the (known) symbols of I in base b, i.e. smsm-1......s2s1s0.

Taking the power series for I in powers of k:-

I = t0 + k(t1 + k(t2 + .... + k(tn-1 + ktn)...))

it can be seen that the integer division of I in base b by k in base b has a remainder t0 in base b which can be converted into a single symbol in base k.

The result of the division is

t1 + k(t2 + .... + k(tn-1 + ktn)...)

which can be iteratively divided by k to give successively t1, t2, .... tn with the iteration stopping when the remainder of the division is zero.

This algorithm does not need to know how many significant symbols there are in either the base b or base k representation.

Example: Convert 5510 to base 2

k = 2, b = 10, k10 = 210
Stage 0: div10 = 5510
Stage 1: Rem10 of 5510 / 210 = 1: t0 = 1: div10 = div10 / 210 = 5510 / 210 = 2710
Stage 2: Rem10 of 2710 / 210 = 1: t1 = 1: div10 = div10 / 210 = 5510 / 210 = 1310
Stage 3: Rem10 of 1310 / 210 = 1: t2 = 1: div10 = div10 / 210 = 5510 / 210 = 610
Stage 4: Rem10 of 610 / 210 = 1: t3 = 0: div10 = div10 / 210 = 5510 / 210 = 310
Stage 5: Rem10 of 310 / 210 = 1: t4 = 1: div10 = div10 / 210 = 5510 / 210 = 110
Stage 6: Rem10 of 110 / 210 = 1: t5 = 1: div10 = div10 / 210 = 5510 / 210 = 010
Done: 5510 = 1101112

Method 3:

This is a second method for converting a positive integer, I, from base b to base k, where division and remainder are available in base b. This method is usable only when there is an upper bound, Imax , on I and the method requires that the powers of k less than Imax are available in base b, i.e. k, k2, k3, .. kr < Imax and kr+1> Imax. The method can be understood with respect to the power series representation of I:-

I = t0k0 + t1k1 + t2k2 + t3k3 + ............. + trkr

where the power series has no terms in kr+1 because Imax of the series is less than kr+1.

Integer division of I in base b by kr gives tr as a result, which can be converted into a symbol in base k.

The remainder is t0k0 + t1k1 + t2k2 + t3k3 + ............. + tr-1kr-1 which can be iteratively divided by kr - 1, kr - 2,...k to yield successively the other symbols of T.

The advantage of this method is that it yields the t symbols in the order trtr-1.......t2t1t0.

Example: Convert 5510 to base 2

k = 2, b = 10, k10 = 210, k310 = 810, k410 = 1610, k510 = 3210, k610 = 6410, ....,
Stage 0: div10 = 5510
Stage 1: t5 = div10 / k510 = 5510 / 3210 = 110 : div10 = Rem10 of 5510 / 3210 = 2310
Stage 2: t4 = div10 / k410 = 2310 / 1610 = 110 : div10 = Rem10 of 2310 / 1610 = 710
Stage 3: t3 = div10 / k310 = 710 / 810 = 010 : div10 = Rem10 of 710 / 810 = 710
Stage 4: t2 = div10 / k210 = 710 / 410 = 110 : div10 = Rem10 of 710 / 410 = 310
Stage 5: t1 = div10 / k110 = 310 / 210 = 110 : div10 = Rem10 of 310 / 210 = 110
Stage 6: t0 = div10 / k010 = 110 / 110 = 110 : div10 = Rem10 of 110 / 110 = 010
Done: 5510 = 1101112

Choice of Method

The choice of which method to use for a conversion is determined primarily by the number system, source or target, in which arithmetic operations can be performed. For most people, arithmetic in base 10 can be performed, so that for manual conversions, method 1 is used when converting into base 10 from another base, but method 2 when converting from base 10.

For computers, only arithmetic in base 2 is available and so method 1 is used when the target number system is base 2, but method 2 when the source number system is base 2.

The decision between method 2 and method 3 is more open to choice, but method 2 is more efficient in the number of operations and is therefore preferable.

For conversions between number systems where arithmetic in neither target or source is available, computers, and most people, will convert via a third number system in which arithmetic is available,

manual conversion on 3516 to tertiary (base 3):

use method 1 to take 3516 to 13910
then method 2 to take 13910 to 120113

k = 6, b = 10, kb = 610

Done: 3516 = 13910
