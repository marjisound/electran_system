Fixed Width Binary Representations of real numbers

Two forms of real numbers are used in computer systems: fixed point and floating point.

Fixed Point Numbers
An n-bit integer holds the n bit symbols to the left of the binary point, i.e. sn ... s1 s0

An n-bit fixed point real number holds the n bit symbols surrounding the binary point, i.e.
sn-m-1 ... s1 s0 s-1 s-2 ... s-m

where the size of m is normally determined by the software, not by the hardware.

These bit symbols are stored such that s-m goes into bit position 0, s-m+1 into bit position 1, etc.

Because m is fixed by the software, the position of the binary point does not have to be explicitly recorded, but is built into the software.

Operations are identical to the integer operations above: integers are really just fixed point numbers with m = 0.

Fixed point numbers may be signed or unsigned.

Conversion from decimal real number to binary real number.

The integer part of the decimal real number is converted in the normal way to binary.

The conversion of the fractional part of the decimal real number follows from the following result:-

The fractional part of a real number represented in base b is

F = s-1b-1 + s-2b-2 + ............. + s-mb-m + ...

Multiplying F by b =

bF = s-1b0 + s-2b-1 + ............. + s-mb-m-1 + ...

and the integer part of bf is just s-1.

Thus repeated multiplication of F by b gives the symbols s-1, s-2, s-3, s-4 in turn.

Thus to get the fractional 0.49768510 into a binary fraction, it just necessary to keep multiplying by 2:-

0.49768510 * 2 = 0.99537010: s-1 = 0
0.99537010 * 2 = 1.99037410: s-2 = 1
0.99037410 * 2 = 1.98074810: s-3 = 1
0.98074810 * 2 = 1.96149610: s-4 = 1
0.96149610 * 2 = 1.92299210: s-5 = 1
0.92299210 * 2 = 1.84598410: s-6 = 1

And so on...

0.49768510 = 0.011111...

Thus to convert 102.49768510 to binary fixed point with 3 fractional bits.

Convert 102 to 01100110 and 0.49768510 to 0.011111

To give 01100110.011111 and truncate to 3 binary digits
